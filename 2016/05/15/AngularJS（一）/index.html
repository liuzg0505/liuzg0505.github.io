AngularJS（一）

1. 学习之前的几个究极问题

1.1 AngularJS 是啥啊？

tip: angular 1.x 版本都是 angularjs.net, 往后版本都是叫angular, 这里往下说的都是针对1.6.6版本。

咳~ 简单的说，AngularJS，就是通过自身的一些内容、指令、表达式，对html代码进行功能的增强。

稍微具体点说呢，AngularJS, 简称 ng 。

其实啊，AngularJS就是一个MVC框架，也可以说是MVVM，或者MVP。

其他前端框架： Vue 、 Avalon 、 React 、 BackBone 、 KnockoutJS

1.2 AngularJS 能干啥啊？

特点：angularJS简化了所有的dom操作

当项目中涉及到很多繁琐但简单的DOM操作的时候，我们可以使用angularJS来开发，这样可以提高开发效率.

1.3 为嘛要用AngularJS？

  • 开发效率高
  • 可以用AngularJS实现单页面应用（spa），做到媲美原生app的效果
  • 单页面应用，可以实现普通多页面应用没法实现的效果

2. SPA - 单页应用程序

SPA: single page application 不是说水疗按摩那SPA, 这里是指 单页面应用程序

介绍：

单页Web应用（single page application，SPA），就是只有一个Web页面的应用，是加载单个HTML页面，并在用户与应用程序交互时动态更新该页面的Web应用程序。

单页面应用又被称为 轻客户端

2.1 优势

  • 减少请求次数，提升用户体验
  • 可以达到媲美原生app的效果
  • 可以实现普通多页面无法实现的功能，比如网易云音乐网站的播放效果

2.2 劣势

  • 首次加载内容过大，等待时间较长
  • 单页面应用不利于做SEO

2.3 主要技术点

  • ajax

  • 锚点的使用（window.location.hash #）

  • hashchange 事件

    window.onhashchange : 地址栏hash值改变时触发的事件，就是锚点变化的时候触发

2.4 实现思路

  • 监听锚点值变化的事件，根据不同的锚点值，请求相应的数据
  • 锚点（#）原本用作页面内部进行跳转，定位并展示相应的内容
  • NG中，锚点被用作请求不同资源的标识，请求数据并展示内容

3. MVC MVP MVVM

3.1 MVC

模型视图控制器 M --> V --> C --> M 单向交互
Model: 所有包含数据的代码，Model层负责提供数据
View: 视图：主要负责展示数据
Controller: 负责从视图读取数据，控制用户输入，并向模型发送数据，是model和view的桥梁

优势：代码分离（视图代码、控制器代码），职责分离，解耦

目的：解决应用程序展示结构、业务逻辑之间的紧耦合关系，实现模块化和复用

提高了代码的结构和可维护性，但是不会提高代码执行的效率

3.2 MVP

M <--->P<--->V

P: Presenter, 将controller的功能进行了加强，和V和M都是双向交互，使view和model完全没有关系，这里面的代码需要自己写。

3.3 MVVM

VM: View-Model, 相当于就是presenter, 只是这是自动的双向数据绑定，里面的代码不需要自己写，可以理解为就是$scope

3.4 Angular说自己是 MVW

Model, View, Whatever (MV啥都行，可以说MVC, 也可以说MVP，也可以说MVVM)

5. 说说指令 (directive)

这里先只说几个常用指令，其他的可以参考AngularJS官方文档 。

  • AngularJS 有一套完整的、可扩展的、用来帮助 Web 应用开发的指令集
  • 指令是DOM元素上的一些标记，让NG给DOM元素添加一些特殊的行为
  • 指令包含：内置指令 和 自定义指令

5.1 指令是什么

  • 将前缀为 ng- 的属性称之为指令，其作用是为DOM元素绑定数据、添加事件 等
  • 指令的值是一个 表达式

5.2 常用指令 (下面还有其他一些指令介绍)

ng-app

  • 作用：该指令用来启动一个AngularJS应用
  • 理解：指定AngularJS应用程序管理的边界，只有在ng-app内部的指令才会起作用
  • 解释：

ng-model

  • 作用：绑定数据，在 input/select/textarea 标签中使用 （只用于表单元素, 表示值和数据模型里面的值双向绑定）
  • 说明：

ng-click

  • 作用：用来指定DOM元素被点击时执行的事件
  • 语法：ng-click="expression" (值是表达式或函数调用)

5.3 补充知识：angular中指令的使用方式

还可以用以下的方式：

8.3.2 控制器的作用

  • 初始视图中使用的数据，是存储数据的容器
  • 通过$scope对象把数据模型或函数行为暴露给视图
  • 监视模型的变化，做出相应的逻辑处理

每个控制器 都有自己的数据模型！

如果控制器之间进行嵌套，那么在找数据的时候，如果当前的$scope中没有找到，就会去上一级控制器中查找

如果在双向绑定的input元素中，文本框为空，那么对应的数据值会是undefined

  • 2 在控制器中暴露 数据模型（数据和行为），在视图中通过指令或表达式来使用

10. 解决页面加载时插值表达式的闪烁问题

10.1 ng-bind 指令 (不是很好用)

ng-bind=“name”指令可以做单向绑定，
在angular加载完成之后，会找到数据模型中的数据，将数据作为当前标签中的内容，标签中的内容会被替换掉。

10.2 ng-cloak 指令 (常用方法)

ng-cloak指令，会在angular加载完成之后，被移除。 需要配合css样式来实现解决插值表达式闪烁问题，

  • 当angular加载完成之前，由于ng-cloak的存在，css样式生效，会将元素隐藏
  • 当angular加载完成之后，会将所有ng-cloak移除，这样，css样式就失效， 元素就显示出来了
